# Architecture Decision Record (ADR) Template

Use this template for major technical decisions (e.g., choosing a new framework, changing the database schema significantly). Do not use this for minor implementation details.

## Context
What is the problem or situation that requires a decision? Keep it brief.

## Decision
What is the specific change or technical choice being made?

## Consequences
What are the expected impacts? (e.g., "Requires all developers to install X", "Improves query speed but adds complexity to the build step").
